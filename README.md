# Teotl V4 — Nightmare Mode

> *A browser-based psychological-horror videogame powered by Rust/WASM and TypeScript.*

Teotl V4 — Nightmare Mode is an atmospheric, browser-native horror experience set in a
liminal pre-Columbian ritual complex that exists outside normal time. The game couples a
**Rust → WebAssembly core** (particle simulation, nightmare escalation state machine, world
graph, NPC and quest systems) with a **TypeScript web host** (Canvas 2D rendering, Web Audio
API, psychological-horror UI effects) to deliver deep environmental dread through glitch
aesthetics, procedural audio, and an ever-escalating *Nightmare Mode* that bends the
interface itself into a tool of psychological horror.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Core engine / game logic | Rust (compiled to WebAssembly via **wasm-pack**) |
| Web host / UI / rendering glue | TypeScript + **Vite** |
| Audio | Web Audio API |
| Rendering | HTML5 Canvas 2D (+ planned WebGL upgrade) |
| Content data | YAML / JSON |
| CI | GitHub Actions |

## Quick Start

### Prerequisites

- [Node.js](https://nodejs.org/) ≥ 20
- [Rust](https://rustup.rs/) stable + `wasm32-unknown-unknown` target
- [wasm-pack](https://rustwasm.github.io/wasm-pack/)

```bash
rustup target add wasm32-unknown-unknown
curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh
```

### 1. Build the WASM crate

```bash
cd crates
wasm-pack build teotl_wasm --target web --out-dir ../web/src/wasm
```

> **Note:** The WASM crate is a placeholder. The TypeScript host runs in
> stub mode until the crate is built.

### 2. Install web dependencies

```bash
cd web
npm ci
```

### 3. Start the development server

```bash
npm run dev
# → http://localhost:3001
```

### 4. Production build

```bash
npm run build
# Output: web/dist/
```
